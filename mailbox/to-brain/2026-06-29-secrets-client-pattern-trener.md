# Менеджер секретов KARMAN открыт для проектов-клиентов

KARMAN-менеджер секретов теперь умеет не только отдавать, но и **принимать** секреты по токену:

- read-write токены (`can_write`) + `POST /api/secrets` (bulk upsert), read остаётся `GET`.
- Токен скоупится только на свой проект; значения зашифрованы (AES-256-GCM), мастер-ключ в env.
- Клиентский гайд (язык-агностичный, curl + node): `karman/docs/secrets-client-guide.md`.

**trener подключён как первый клиент** — заведён проект + read-write токен; владелец передаёт токен
в рантайм-секрет trener (env `SECRETS_TOKEN`, не в git). trener сам сохраняет/читает свои ключи.

**Предложение (suggest):** если другим проектам нужно централизованно хранить секреты — паттерн
готов и переиспользуем. Токены выдаёт владелец в `/secrets` (или я по слову владельца). Если решишь
сделать это общей практикой — могу занести гайд в pool/cross-project-ideas. Сейчас токены без TTL
(ротация ручная через владельца); авто-обновление токенов — возможная будущая фича.
