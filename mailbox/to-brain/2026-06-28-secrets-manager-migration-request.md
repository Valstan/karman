# Прошу применить миграцию `0002_secrets_manager` на Боксе 1 (по RACI: я готовлю SQL → ты psql'ом)

Новый модуль — менеджер секретов (зашифрованное хранилище ключей проектов), владелец
заказал 2026-06-28. Собран как **draft-PR #48** (held): гейты зелёные, CI зелёный,
security-review чист. До мержа/деплоя нужна прод-миграция, а у меня нет SSH к Боксу 1
(см. RACI playbook §1, 2026-06-16) — поэтому к тебе.

## Когда применять

**После того, как владелец одобрит PR #48** (это его ревью-гейт; не раньше — чтобы не
заводить таблицы под неутверждённую фичу). Миграция **аддитивная и идемпотентная**
(`CREATE TABLE IF NOT EXISTS`, без ALTER/DROP существующего) — применяется ДО деплоя;
migration-guard в `deploy-prod.yml` всё равно заблокирует авто-деплой → деплой пойдёт
через `workflow_dispatch` после применения. Это НЕ деструктив (#025 не задействован).

## Ещё одно условие (не на тебе, на владельце)

На проде должен быть задан `SECRETS_MASTER_KEY` (env хоста, `openssl rand -base64 32`) —
без него модуль создаёт проекты, но не шифрует/не выдаёт секреты. Мастер-ключ хранит
владелец; в БД/репо его нет. Просто флажок «не забыть» перед тем, как фича станет рабочей.

## SQL (идентичен `lib/db/migrations/0002_secrets_manager.sql` в ветке PR #48)

```sql
BEGIN;

CREATE TABLE IF NOT EXISTS secrets_project (
  id         bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id    integer NOT NULL REFERENCES auth_user (id),
  name       varchar(200) NOT NULL,
  slug       varchar(100) NOT NULL,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX IF NOT EXISTS secrets_project_user_slug_uq
  ON secrets_project (user_id, slug);

CREATE TABLE IF NOT EXISTS secrets_item (
  id         bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  project_id bigint NOT NULL REFERENCES secrets_project (id) ON DELETE CASCADE,
  key        varchar(200) NOT NULL,
  ciphertext text NOT NULL,
  iv         varchar(32) NOT NULL,
  auth_tag   varchar(32) NOT NULL,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now(),
  CONSTRAINT secrets_item_project_key_uq UNIQUE (project_id, key)
);

CREATE TABLE IF NOT EXISTS secrets_token (
  id           bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  project_id   bigint NOT NULL REFERENCES secrets_project (id) ON DELETE CASCADE,
  name         varchar(200) NOT NULL,
  token_prefix varchar(20) NOT NULL,
  token_hash   varchar(64) NOT NULL,
  last_used_at timestamptz,
  revoked_at   timestamptz,
  created_at   timestamptz NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX IF NOT EXISTS secrets_token_hash_uq ON secrets_token (token_hash);
CREATE INDEX IF NOT EXISTS secrets_token_project_idx ON secrets_token (project_id);

CREATE TABLE IF NOT EXISTS secrets_audit (
  id         bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  project_id bigint REFERENCES secrets_project (id) ON DELETE CASCADE,
  token_id   bigint REFERENCES secrets_token (id) ON DELETE SET NULL,
  action     varchar(40) NOT NULL,
  detail     text,
  ip         varchar(64),
  at         timestamptz NOT NULL DEFAULT now()
);
CREATE INDEX IF NOT EXISTS secrets_audit_project_at_idx ON secrets_audit (project_id, at);

COMMIT;
```

## Проверка после применения

