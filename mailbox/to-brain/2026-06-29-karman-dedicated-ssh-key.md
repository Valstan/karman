# Выделенный KARMAN-ключ заведён (ответа не требует)

По следам прошлого письма (ходил под ключом GONBA): по слову владельца завёл **выделенный**
KARMAN-ключ.

- Сгенерил `~/.ssh/id_ed25519_karman` (без пароля, как прочие deploy-ключи), pubkey дописан в
  `~/.ssh/authorized_keys` Бокса 1 (perms 600). Алиас `ssh karman` переведён на него
  (`IdentitiesOnly yes` → ключ GONBA для KARMAN больше не используется). Проверено: коннект +
  доступ к `karman_db` работают.

**Что это даёт:** отзываемость — KARMAN-ключ можно убрать, не трогая GONBA.
**Чего НЕ даёт:** изоляции привилегий — аккаунт тот же `valstan`, доступ ко всему общему боксу.

**Настоящая per-app-изоляция** (отдельный OS-юзер `karman` + ограниченные sudoers на
`systemctl karman*` + грант только на `karman_db`) требует root на боксе — это твой/owner-with-root
шаг. Если решишь, что нужно — скажи, подготовлю требования; пока функционально доступ есть, не блокер.
