# #035 у KARMAN: инвентарь, план, вопросы

## 1. Инвентарь полей поиска (#022 — все потребители)

Поверхность маленькая (3 списочные страницы):

| # | Где | Поля haystack | Сейчас |
|---|---|---|---|
| 1 | `components/app/credits-table.tsx` | name, bankName | client-side **substring** (`includes`), один токен, без нормализации/подсветки + select-фильтр статуса |
| 2 | `components/app/documents-table.tsx` | title, documentType, documentNumber, issuingAuthority, categoryName | то же + select статус/категория |
| 3 | `components/app/banks-table.tsx` | — | поиска нет (список мал) — подключу shared-модуль бесплатно |
| 4 | Селекты форм (credit/document/payment-dialog) | банк, категория, кредит | radix Select, фиксированные короткие списки, БЕЗ typeahead — combobox с searchMatch только когда списки вырастут |
| 5 | Серверный поиск/пагинация | — | ещё не существует (в PENDING, преждевременно: вся БД 11 MB) |

**Хорошая новость:** prefix-only у меня нет — Уровень 1 (substring в любом месте) уже
стоит в обоих полях с PR #11. Нет: многотокена AND, нормализации (ё↔е, номера `240-1`≡`2401`),
Ур.2 subsequence, Ур.3 fuzzy, RU↔EN, подсветки, ранжирования, shared-модуля.

## 2. Фазовый план

- **Ф0 — shared-модуль** `lib/search/tiered-search.ts`: `searchMatch`/`rankMatches` по спеку
  (нормализация → многотокен AND → substring → subsequence → fuzzy(Левенштейн) → ранг →
  диапазоны подсветки → RU↔EN на нуле результатов) + vitest-тесты. Чистый JS, без БД.
- **Ф1 — подключение**: credits-table + documents-table (+ banks-table) через единый модуль,
  подсветка совпадений, пометка «похожие» на fuzzy-результатах. Данные уже целиком на
  клиенте (списки маленькие) — все уровни в JS, debounce не нужен.
- **Ф2 — комбобоксы форм**: отложено, пока списки селектов короткие (поведение единое
  получат из того же модуля при переходе на combobox).
- **Ф3 — серверный поиск/пагинация** (когда записей станет много): Ур.1 → `ILIKE '%q%'`,
  Ур.3 → `pg_trgm` similarity + GIN; debounce + min-chars ≥2; probe латентности на реальном
  объёме (#020) до выката. Связан с пунктом PENDING «серверный поиск» — поглотил его.

**Ф0+Ф1 беру в работу сразу** (текущая сессия, отдельным feat-PR). Ф3 остаётся в
`docs/PENDING_FOLLOWUPS.md` с метками старения — не потеряется (#033).
