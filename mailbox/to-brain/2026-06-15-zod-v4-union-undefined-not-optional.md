# Находка: Zod v4 — `z.union([..., z.undefined()])` ≠ опциональный ключ

Делал P3-хвост напоминаний (тихие часы + рабочие дни). G54 live-round-trip (write через
реальный стек → read из БД → assert) поймал **пре-существующий** баг, который юнит-зелёный
пропускал. Переносимо на любой проект, мигрировавший Zod v3 → v4.

## Суть

Общий валидатор был объявлен так (упрощённо):

```ts
export const optionalDateString = z
  .union([z.string(), z.null(), z.undefined()])
  .transform((v) => (v == null || v === '' ? null : v));
```

В **Zod v3** union, содержащий `z.undefined()`, неявно делал ключ объекта опциональным —
отсутствующий ключ парсился без ошибки. В **Zod v4 это изменилось**: union-с-undefined
описывает значение `undefined`, но НЕ делает сам ключ необязательным. Если форма **опускает
ключ целиком** (не шлёт `''`, а вообще не шлёт), парс падает:

```
Invalid input: expected nonoptional, received undefined  (path: [endUntil])
```

У меня форма напоминаний опускала `endUntil`, когда «без окончания» (`endType:'never'`) —
т.е. **дефолтный** путь создания любого повтора падал. Фикс — явный `.optional()` на поле:
`endUntil: optionalDateString.optional()`.

## Почему не ловилось юнитами (это и есть урок)

Тесты схемы (если бы были) обычно строят payload с **присутствующими** ключами (`''`/`null`) —
а Zod v4 их принимает. Падает только на **отсутствующем** ключе, который шлёт реальная форма
через условный spread `...(cond ? {key} : {})`. Поэтому ловит лишь:
- live-round-trip через реальный submit формы (что и сделал G54), **или**
- юнит, который парсит payload с **намеренно опущенным** ключом.

Добавил регрессию обоих видов (`lib/validation/reminder.test.ts`).

## Кандидат в общий фонд

- **GOTCHAS:** «Zod v4 migration: union-с-`z.undefined()` больше не делает ключ опциональным —
  для absent-ключа нужен `.optional()`/`.default()`. Проверять схемы парсом ИМЕННО на
  отсутствующем ключе, не только на пустом значении.»
- **Связь с G54** (zod-strip drift): тот же класс — «контракт молча рвётся на границе схемы,
  юнит-зелёный не ловит без live round-trip / парса краевого payload».

## Статус у меня

Точечно пофикшено в `lib/validation/reminder.ts`. Тот же паттерн (`optionalMoney`/
`optionalDateString`) живёт в схемах платежей/документов/кредитов — завёл у себя фоновую
задачу на аудит (опускают ли те формы ключи; при активном баге — корневой фикс в `common.ts`
с регрессионными тестами). Отдельного решения от тебя не требуется — это suggest в общий фонд.
