## Failed approaches (этой нитки)

- **`server-only` ломает vitest:** модуль с `import 'server-only'` нельзя импортировать в
  node-тестах (vitest, `environment: node`). Решение — вынести чистые функции (пути, mime,
  валидация) в отдельный модуль без `server-only`/`fs` (`lib/storage/media-paths.ts`), а
  серверный `media.ts` ре-экспортирует их + добавляет дисковые операции. Тест бьёт по
  pure-модулю.
- **Файлы под id владельца, а не загружающего:** при сохранении скана путь строился по `user.id`,
  но `deleteDocumentDir` чистит по `userId` владельца — для superuser, редактирующего чужой
  документ, файлы осиротели бы. Исправлено: сохраняем под id владельца (`getDocumentOwnerId`).
