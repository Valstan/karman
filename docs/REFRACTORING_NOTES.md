# Refactoring Notes

## Что уже сделано (реально подтверждено)

- Удалён ранний Django backend из рантайма.
- Удалены связанные с ним systemd unit-файлы и служебные настройки.
- Упрощён SPA-роутинг для работы напрямую с root path.
- Обновлён nginx fallback для `index.html`.
- Добавлен `api/server.js` на Express + PostgreSQL как основной API.
- Подключена аутентификация через `auth_user`, с cookie-сессией `karman_session`.
- Обновлён фронт для реальных запросов (`dashboard`, `banks`, `credits`, `payments`, `documents`).
- Добавлен API service (`karman-api.service`) и базовые deployment скрипты.

## Текущий статус по качеству

- Архитектура стабильная для базовой эксплуатации.
- Требуется дисциплина по документации: после каждого изменения API/фронта обновлять описание маршрутов и поведение.

## Что планируется улучшать

- Переносить роуты SPA на ленивую подгрузку модулей (уменьшение размера первого bundle).
- Добавить нормализацию и валидацию request payload в API.
- Добавить типизированный API client слой для фронта (или вынести все типы в общий пакет).
- Ввести единый формат ошибок и централизованный middleware-обработчик бизнес ошибок.
- Добавить e2e проверку критических сценариев (вход, создание/редактирование кредита, смена статуса платежа).
- Рассмотреть `favicon.ico` и единые визуальные токены для стабильного продакшн-UI.

## Передача контекста между сессиями нейросети

- Для непрерывности работ между сессиями веди `docs/AI_SESSION_CONTINUITY.md`.
