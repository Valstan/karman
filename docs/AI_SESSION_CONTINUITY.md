# AI Session Continuity (для передачи контекста между нейросетями)

Цель файла — чтобы каждая новая сессия быстро понимает состояние проекта без повторного разбора всей базы кода.

## 1) Что есть сейчас

- Проект называется **KARMAN**.
- Технологический стек:
  - `frontend/` — React + TypeScript + Vite + Ant Design.
  - `api/` — Node.js (Express) + PostgreSQL.
  - `nginx` — статика из `frontend_dist/` + proxy к API на `127.0.0.1:8080`.
- Авторизация: сессия через `HttpOnly` cookie `karman_session`, данные пользователей из `auth_user`.
- Роутинг SPA:
  - `/dashboard`, `/credits`, `/payments`, `/banks`, `/documents`, `/login`.
- Основные API-маршруты:
  - `/api/v1/auth/*`
  - `/api/v1/dashboard/summary/`
  - `/api/v1/credits/credits/`
  - `/api/v1/credits/payments/`
  - `/api/v1/credits/banks/`
  - `/api/v1/documents/`
  - `/api/health`
- Документация по общей архитектуре и операциям поддерживается в:
  - `docs/ARCHITECTURE.md`
  - `docs/OPERATIONS.md`
  - `docs/REFRACTORING_NOTES.md`

## 2) Проверки среды (актуальны на момент последнего обновления)

- Проверка sudo без пароля: `sudo -n true` — **успешно**, пароль не требуется.
- Для диагностики проекта используем:
  - Базовая сборка frontend.
  - Базовые curl-проверки API из `docs/OPERATIONS.md`.

## 3) Старт новой сессии (рекомендуемый порядок)

1. Считать этот файл целиком.
2. Коротко проверить `git status`.
3. Считать последнюю запись в секции `История изменений сессий`.
4. Проверить, нет ли актуальных блокеров в `docs/ARCHITECTURE.md`, `docs/OPERATIONS.md`, `docs/REFRACTORING_NOTES.md`.
5. Продолжить задачу.
